# 関連付け操作

## 概要

前提知識追加機能実装に向け、tinkerなどでいろいろ確認。

## サンプルデータ

Trace:

| id | title      |
| -- | ---------- |
| 1  | PHP Array  |
| 2  | Collection |
| 3  | map()      |

Collectionの前提知識にPHP Arrayを追加したい。

## まずはcreate()で

```php
<?php
App\Models\TraceRelation::create(['from_trace_id' => 1, 'to_trace_id' => 2, 'relation_type' => App\Enums\TraceRelationType::PREREQUISITE])
```

## Livewireでやりたいのは

```php
<?php
$currentTrace->addPrerequisite($anotherTrace);
```

みたいな書き方をしたい。

```php
<?php
$collection = Trace::find(2);

$phpArray = Trace::find(1);

$collection->addPrerequisite($phpArray);
```

## HasManyリレーションからcreate()

```php
<?php
$collection->outgoingRelations()
```

このとき返ってくるのは`HasMany` = リレーションオブジェクト。(SQL発行前)

`outgoingRelations()`:

```php
<?php
public function outgoingRelations() :HasMany
{
    return $this->HasMany(TraceRelation::class, 'from_trace_id');
}
```

つまり、`$trace->outgoingRelations()`は`from_trace_id = このTraceのid`という意味。


実際に作ってみる:

```php
<?php
$phpArray = Trace::find(1);

$phpArray->outgoingRelations()->create(['to_trace_id' => 2, 'relation_type' => App\Enums\TraceRelationType::PREREQUISITE])
```

=> `from_trace_id`には`$phpArray`のidが自動で入る

## Collection側から追加する場合は？

```php
<?php
$collection->incomingRelations()->create(['from_trace_id' => $phpArray->id, 'relation_type' => App\Enums\TraceRelationType::PREREQUISITE])
```

incomingRelations()は`return $this->hasMany(TraceRelation::class, 'to_trace_id');`

なのでcreate()すると、`'to_trace_id' => $collection->id`を自動で補完してくれる。
