# belongsToMany 

## 概要

中間テーブルを使った多対多リレーション(`many-to-many`)

## 例: PostとTag

1つの投稿に複数のタグが付き、1つのタグが複数の投稿に付くという関係。

`posts`と`tags`の2テーブルだけでは表現できないため、中間テーブル(`pivot table`)が必要になる。

```
posts    tags    post_tag(中間テーブル)
-----    ----    ----------------------
id       id      post_id
title    name    tag_id
```

## 規約(中間テーブル名)

Eloquentは、関連する2モデル名をアルファベット順にスネークケース・単数形で連結したテーブル名を期待する。

- Post + Tag => `post_tag`
- Trace + Tag => `tag_trace`

```php
<?php

class Post extends Model
{
    public function tags()
    {
        return $this->belongsToMany(Tag::class);
    }
}

class Tag extends Model
{
    public function posts()
    {
        return $this->belongsToMany(Post::class);
    }
}
```

=> 規約から外れる場合は明示的に指定もできる

```php
<?php
return $this->belongsToMany(
    Tag::class,
    'tagging', // 中間テーブル名
    'post_id', // このモデル(Post)の外部キー
    'tag_id'   // 関連先モデル(Tag)の外部キー
);

```

## belongsToMany()メソッド: 4引数

```php
<?php
belongsToMany(
    関連モデル,
    中間テーブル,
    現在モデル側キー,
    相手モデル側キー
)
```

規約通りのテーブル名なら省略可能:

```php
<?php
// 中間テーブル: tag_trace
public function tags(): BelongsToMany
{
    return $this->belongsToMany(Tag::class); // 第一引数のみ
}

// 内部的にはこれと同じ
return $this->belongsToMany(
    Tag::class,
    'tag_trace',
    'trace_id',
    'tag_id'
);
```

## 基本操作

```php
<?php
// 取得
$post = Post::find(1);
$post->tags;              // タグのコレクション
$post->tags()->get();     // 同じ(クエリとして条件追加も可)
$post->tags()->where('name', 'PHP')->first();

// アタッチ・デタッチ・同期
$post->tags()->attach($tagId);            // 中間テーブルに1行追加
$post->tags()->attach([1, 2, 3]);         // 複数追加

$post->tags()->detach($tagId);            // 中間テーブルから削除
$post->tags()->detach();                  // 全て削除

$post->tags()->sync([1, 2, 3]);           // 指定したIDだけの状態に同期(なければ削除、なければ追加)
$post->tags()->syncWithoutDetaching([4]); // 既存を消さずに追加
$post->tags()->toggle([1, 2]);            // あれば外す、なければ付ける
```

=> `sync()`がよく使用される

=> フォームでチェックボックスでタグを選択させ、選ばれたIDの配列をそのまま渡すだけで中間テーブルが整う
