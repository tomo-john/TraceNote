# 自己参照リレーション

## 概要

初めて自己参照リレーションを実装するにあたり、基礎知識を学習。

### 自己参照とは何か？

普通のリレーションは、`User -> Post`のように異なるテーブル同士の関係。

自己参照は、`User -> User`のように同じテーブル同士が関係を持つことを言う。

### 自己参照(1対多)

これが一番よくある自己参照。

例(組織図):

```
社長
├── 部長A
│     ├── 課長A
│     └── 課長B
└── 部長B
```

テーブル:

| id | name   | parent_id |
| -- | ------ | --------- |
| 1  | 社長   | null      |
| 2  | 部長A  | 1         |
| 3  | 部長B  | 1         |
| 4  | 課長A  | 2         |
| 5  | 課長B  | 2         |

### 自己参照(多対多)

今回使おうとしているやつ。

例(SNSの友達関係):

```
John <-> Pyonkichi
John <-> Mocomoca
Pyonkichi <-> Pyonta
```

usersテーブル:

| id | name      |
| -- | --------- |
| 1  | John      |
| 2  | Pyonkichi |
| 3  | Mocomoca  |
| 4  | Pyonta    |

user_friendテーブル(中間テーブル):

| user_id | friend_id |
| ------- | --------- |
| 1       | 2         |
| 1       | 3         |
| 2       | 4         |

Laravelでは:

```php
<?php
public function friends()
{
    return $this->belongsToMany(
        User::class,
        'user_friend',
        'user_id',
        'friend_id'
    );
}
```

ポイント:

```php
<?php
belongsToMany(
    相手のモデル,
    中間テーブル,
    自分側の外部キー,
    相手側の外部キー
)
```

ちょっとイメージ(traces):

| id | title       |
| -- | ----------- |
| 1  | Collection  |
| 2  | map         |
| 3  | mapWithKeys |

trace_relations:

| trace_id | related_trace_id |
| -------- | ---------------- |
| 1        | 2                |
| 2        | 3                |

```
Collection -> map
map -> mapWithKeys
```

```php
<?php
public function relatedTraces()
{
    return $this->belongsToMany(
        Trace::class,
        'trace_relations',
        'trace_id',
        'related_trace_id'
    );
}
```

=> `$trace->relatedTraces`が使えるようになる
