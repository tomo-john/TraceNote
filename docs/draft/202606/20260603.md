# Eloquent

## 概要

LravelのORM(Object-Relational Mapping)

1つのDBテーブル = 1つのモデル

SQLを書かずにPHPのオブジェクトとしてDBを操作できる仕組み

## 基本のCRUD

```php
<?php
// Create
$post = Post::create(['title' => 'Hello', 'body' => '...']);

// または
$post = new Post;
$post->title = 'Hello';
$post->save();

// Read
$post = Post::find(1);          // 1件、なければnull
$post = Post::findOrFail(1);    // なければ404例外
$posts = Post::all();           // 全件
$posts = Post::where('published', true)->get();

// Update
$post = Post::find(1);
$post->title = '更新後';
$post->save();

// or
Post::where('id', 1)->update(['title' => '更新後']);

// Delete
$post = Post::find(1);
$post->delete();

// or
Post::destroy(1); // IDで削除
```

## Query Builder

EloquentモデルはLaravelのQuery Builderをそのまま使える。

```php
<?php
$posts = Post::where('published', true)
    ->orderBy('created_at', 'desc')
    ->limit(10)
    ->get();

$count = Post::where('author_id', 5)->count();

$post = Post::where('slug', 'hello-world')->first();
```

## リレーション

```php
<?php
class User extends Model
{
    public function posts()
    {
        return $this->hasMany(Post::class); // 1人のUserが複数のPost
    }
}

class Post extends Model
{
    public function author()
    {
        return $this->belongsTo(User::class, 'user_id'); // Postは1人のUserに属する
    }

    public function tags()
    {
        return $this->belongsToMany(Tag::class); // 多対多(中間テーブルpost_tag)
    }

    public function comments()
    {
        return $this->hasMany(Comment::class);
    }
}
```

```php
<?php
// 使い方
$user = User::find(1);
$user->posts;          // プロパティのようにアクセス(リレーションメソッド名)
$user->posts()->where('published', true)->get(); // クエリとして絞り込みも可能
```

主なリレーションの種類:

- hasOne / hasMany: 1対1 / 1対多
- belongsTo: 多対1(外部キーを持つ側)
- belongsToMany: 多対多(中間テーブル経由)
- hasManyThrough: 中間モデルを経由した1対多
- morphTo / morphMany: ポリモーフィックリレーション
    - (複数の異なるモデルに共通の関連を持たせる、例:コメントが投稿にも商品にも付けられる)
