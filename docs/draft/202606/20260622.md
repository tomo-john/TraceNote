# リレーションめも

## 概要

`trace_relations`テーブル / `TraceRelations`モデル

### trace_relations イメージ

| id | from_trace_id | to_trace_id | relation_type |
| -- | ------------- | ----------- | ------------- |
| 1  | 2             | 8           | A             |
| 2  | 8             | 5           | B             |


Traceが仮に、`2: PHP Array`, `8: Collection`, `5: map()`とする。

矢印では、`PHP Array => Collection => map()`

### Traceモデルのメソッド

- outgoingRelations(): 自分から出ていく矢印
- incomingRelations(): 自分に入ってくる矢印

=> `trace_relations`テーブルでイメージする

```php
<?php

    public function outgoingRelations() :HasMany
    {
        return $this->HasMany(TraceRelation::class, 'from_trace_id');
    }

    public function incomingRelations() :HasMany
    {
        return $this->HasMany(TraceRelation::class, 'to_trace_id');
    }
```

### Step1: まずは普通の1対多で考える

```
Trace 1 --- N TraceRelations
```

### Step2: TraceRelationからまたTraceへ戻る

TraceRelations側では:

```php
<?php

    public function fromTrace() :BelongsTo
    {
        return $this->belongsTo(Trace::class, 'from_trace_id');
    }

    public function toTrace() :BelongsTo
    {
        return $this->belongsTo(Trace::class, 'to_trace_id');
    }
```

### なので

```
Trace
 ↓ hasMany
TraceRelation
 ↓ belongsTo
Trace
```

```
Trace -> TraceRelations -> Trace
```

```
Trace <-> Trace
```

Traceモデル:

```php
<?php
outgoingRelations()
incomingRelations()
```

TraceRelationsモデル:

```php
<?php
fromTrace()
toTrace()
```

ここまでを押さえた上で、

```php
<?php
prerequisiteTraces()
childTraces()
relatedTraces()
```

を作る。

---

### おさらい

- `prerequisiteTraces()` => 前提知識のTraceを取得
- `childTraces()` => 子(詳細)知識のTraceを取得
- `relatedTraces()` => 関連知識のTraceを取得

### prerequisiteTraces()

前提知識 = 自分に向く矢印 = `incommingRelations()`

`incommingRelations()`:

```php
<?php
    public function incomingRelations() :HasMany
    {
        return $this->HasMany(TraceRelation::class, 'to_trace_id');
    }
```

=> なんで`to_trace_id`なのか？


| id | from_trace_id | to_trace_id | relation_type |
| -- | ------------- | ----------- | ------------- |
| 1  | 2             | 8           | A             |
| 2  | 8             | 5           | B             |

- 2 = PHP Array
- 8 = Collection
- 5 = map()

Collection(id=8)から見た時、前提知識はPHP Array(id=2) => `from_trace_id = 2`がほしい

ここで大事なのは、欲しいものとTraceRelationレコードを探す条件を分けて考えること。

まず探すべきレコードは？ => Collection(id=8)に向かってくる矢印を探したい。

```sql
where to_trace_id = 8
```

に一致するレコード。

すると、

| id | from_trace_id | to_trace_id |
| -- | ------------- | ----------- |
| 1  | 2             | 8           |

が見つかる。

これが、

```php
<?php
    public function incomingRelations() :HasMany
    {
        return $this->HasMany(TraceRelation::class, 'to_trace_id');
    }
```

の意味。

見つかった、TraceRelation: `2 -> 8`から`$relation->fromTrace`を辿る。

すると、`2 = PHP Array`が取得できる。

- `incomingRelations()`は自分に向かってくる矢印を持つTraceRelationを集める
- `outgoingRelations()`は自分から出ていく矢印を持つTraceRekationを集める

`prerequisiteTraces()`の構想:

- Traceから前提知識のTraceを取得したい
- 呼び出すのはTrace、返ってくるのもTrace(複数)

```php
<?php
$collection->incomingRelations->where('relation_type', 'prerequisite')->map(fn ($relation) => $relation->fromTrace)
```

=> Traceモデルが格納されたCollectionが返る

=> `incomingRelations`はプロパティアクセスなので一度全件取得してしまう

```php
<?php
$collection->incomingRelations()->where('relation_type', 'prerequisite')->with('fromTrace')->get()->map(fn ($relation) => $relation->fromTrace)
```
