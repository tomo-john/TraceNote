# VMware Workstation

## 概要

Broadcom社(旧VMware社)が開発するデスクトップ向けの仮想化ソフトウェア。

## 主な特徴

- マルチOS実行: ホストOSと異なるゲストOSを同時稼働できる
- スナップショット: VMの状態を保持し、いつでも戻せる
- ネットワーク仮想化: ブリッジ/NAT/ホストオンリーなど柔軟なネットワーク構成

## ネットワーク

- ブリッジ: VMが物理LANに直接参加し、独立したIPを取得
- NAT: ホストPCが通信を代理、VMはプライベートIPを使用
- ホストオンリー: 仮想ネットワーク内のみで通信、外部LANには非公開

### ブリッジ(Bridged)

VMが物理ネットワークに直接参加する。

ホストPCと同じLAN上にある、もう1台の独立したPCのように振る舞う。

VMはホストと同じネットワークセグメントからIPを取得。(ルーターのDHCPから直接もらう)

LAN内の他のPCから直接アクセスが可能。(VM同士の双方向通信できる)

### NAT(Network Address Translation)

VMはホストPCの裏側に隠れた専用のプライベートネットワークに接続。

外部との通信はすべてホストPCが代理(変換)して行う。 => 家庭用ルーターと同じ仕組み

VMは独自のプライベートIPを持つ。インターネットへのアクセスは可能(ホスト経由)

外部(物理LAN側)からVMへ直接アクセスすることはできない。

### ホストオンリー(Host-only)

ホストPCとVMだけで構成される。

外部と完全に隔離された仮想ネットワーク。(物理LANやインターネットに繋がらない)

ホスト <-> VM 間の通信は可能。

| モード   | 仮想スイッチ | ホストアダプタ                       | 接続性                                     |
| -------- | ------------ | ------------------------------------ | ------------------------------------------ |
| ブリッジ | vmnet0       | ホストの物理NIC自体が参加            | 同じLAN内なので普通に到達可能              |
| NAT      | vmnet8       | ホストにvmnet8用の仮想アダプタが生成 | ホストは仮想アダプタ経由でVMに到達可能     |
| HO       | vmnet1       | ホストにvmnet1用の仮想アダプタが生成 | ホストとVMは1対1の閉じたネットワークで直結 |
