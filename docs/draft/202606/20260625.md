# Livewireのプロパティ

## 概要

Array vs Collection(プロパティの型として)

## Livewireの公開プロパティ

リクエストごとにJSONとしてシリアライズされ、ブラウザに送られ、次のアクション実行時に戻ってくる。

この仕組み上でArrayとCollectionでは挙動が異なる。

## Array

メリット:

- シンプルでオーバーヘッドが最小。JSONとの相互互換が直接的。
- フォームの動的な行追加・削除に向いている

デメリット:

- PHPの素のAPI(array_map, array_filterなど)しか使えない

## Collection

メリット:

- map, filterなど豊富なメソッドが使える
- モデルのメソッドやアクセサも使い続けられる

デメリット:

- EloquentのCollectionの場合、毎リクエストでDBへ再クエリが発生
- EagerLoad(`with()`)を忘れるとN+1問題が起きやすい

## Computed

```php
<?php
use Livewire\Attributes\Computed;

class ShowPosts extends Component
{
    public function render()
    {
        return view('livewire.show-posts');
    }

    #[Computed]
    public function posts()
    {
        return Auth::user()->posts; // DBクエリ
    }
}
```

Bladeでは`$this->posts`という形でアクセスする。

## 目安

| 用途                                                       | 使用するもの                   |
| ---------------------------------------------------------- | ------------------------------ |
| ユーザー入力で書き換える配列(フォームの動的フィールドなど) | Arrayプロパティ                |
| 状態として保持しつつ、常に最新のモデル群を扱いたい         | Collection(Eloquent)プロパティ |
| DBから取得して表示するだけ・加工するだけ                   | Computed                       |
