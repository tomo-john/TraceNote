# Laravel 自己参照多対多リレーション基礎

## 概要

自己参照多対多(Self-Referencing Many-to-Many)とは、
同じテーブル同士で多対多の関係を持つリレーション。

代表例

* ユーザー同士のフォロー
* SNSの友達
* 関連商品
* 従業員同士のメンター関係

---

## テーブル構成

### users

| id | name |
| -- | ---- |
| 1  | John |
| 2  | Mike |
| 3  | Tom  |

### user_follow

中間テーブル

| follower_id | following_id |
| ----------- | ------------ |
| 1           | 2            |
| 1           | 3            |

意味

* John → Mikeをフォロー
* John → Tomをフォロー

---

## マイグレーション

```php
Schema::create('user_follow', function (Blueprint $table) {
    $table->foreignId('follower_id')
        ->constrained('users')
        ->cascadeOnDelete();

    $table->foreignId('following_id')
        ->constrained('users')
        ->cascadeOnDelete();

    $table->primary([
        'follower_id',
        'following_id'
    ]);
});
```

## モデル

```php
class User extends Model
{
    //
}
```

---

## followings()

自分がフォローしているユーザー

```php
public function followings()
{
    return $this->belongsToMany(
        User::class,
        'user_follow',
        'follower_id',
        'following_id'
    );
}
```

引数

```php
belongsToMany(
    関連モデル,
    中間テーブル,
    自分側のキー,
    相手側のキー
)
```

---

## followers()

自分をフォローしているユーザー

```php
public function followers()
{
    return $this->belongsToMany(
        User::class,
        'user_follow',
        'following_id',
        'follower_id'
    );
}
```

キーが逆になる。

---

## データ取得

### フォロー一覧

```php
$user = User::find(1);

$user->followings;
```

---

### フォロワー一覧

```php
$user->followers;
```

---

## フォローする

```php
$user->followings()->attach($targetUserId);
```

例

```php
$user->followings()->attach(2);
```

---

## フォロー解除

```php
$user->followings()->detach($targetUserId);
```

---

## 重複を防ぐ

```php
$user->followings()
    ->syncWithoutDetaching([$targetUserId]);
```

既に登録済みなら追加されない。

---

## 件数取得

フォロー数

```php
$user->followings()->count();
```

フォロワー数

```php
$user->followers()->count();
```

---

## eager loading

```php
$users = User::with([
    'followings',
    'followers'
])->get();
```

N+1問題を防げる。

---

## withCount()

```php
$users = User::withCount([
    'followings',
    'followers'
])->get();
```

取得できるカラム

```php
$users[0]->followings_count;

$users[0]->followers_count;
```

## ポイント

* 同じモデル同士を結び付ける
* belongsToMany() を2つ定義する
* キーの向きが逆になる
* 中間テーブルには2つの外部キーを持たせる
* attach()で追加
* detach()で削除
* syncWithoutDetaching()で重複防止
* withCount()で件数取得可能

自己参照多対多は

「同じモデル同士を中間テーブルで結ぶ」

という考え方を理解すると扱いやすくなる。
