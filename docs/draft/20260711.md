# Service Container

## 概要

LaravelのService Containerについて。

## よく見るやつ

```php
<?php
public function __construct(UserService $service)
{
    $this->serice = $service;
}
```

=> `new UserService()`しなくても使用できる

=> Laravelが自動で作って渡してくれる

## __construct

PHPで`__`(アンダースコア2つ)から始まるメソッドはマジックメソッド。

マジックメソッドは、自分で呼ばずにPHPが自動で呼んでくれる。

## DI(依存性注入)

```php
<?php
class UserController
{
    private UserService $service;

    public function __construct()
    {
        $this->service = new UserService();
    }
}
```

これだとControllerがUserServiceを自分で作る。

=> Controllerが作り方まで知ってしまう

```php
<?php
class UserController
{
    public function __construct(
        UserService $service
    ) {
        $this->service = $service;
    }
}
```

=> これがDI

=> DIを実現しているのが`Service Container`
