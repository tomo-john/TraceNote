# Eloquent

## 概要

LaravelのORM(Object Relational Mapper)

## ORMとは

データベースのテーブルをPHPのオブジェクトとして扱える仕組み。

```sql
SELECT * FROM users;
```

これが

```php
<?php
$users = User::all();
```

でいける仕組み。

=> Userクラスがusersテーブルそのものみたいに扱える

## クエリビルダとの違い

クエリビルダ:

```php
<?php
use Illuminate\Support\Facades\DB;

$users = DB::table('users')->where('age', '>=', 20)
                           ->get();
```

- SQLを書きやすくしたもの
- テーブル単位で操作する
- 戻り値はオブジェクトの集合(`stdClass`)

Eloquent:

```php
<?php
$users = User::where('age', '>=', 20)->get();
```

- モデルを操作する
- レコードが`User`オブジェクトになる
- リレーションが使える
- Laravelらしい書き方


| Eloquent            | クエリビルダ              |
| ------------------- | ------------------------- |
| Laravelらしい書き方 | SQL寄り                   |
| モデルを使う        | テーブルを使う            |
| リレーションが強力  | リレーションなし          |
| 保守しやすい        | 少し高速                  |
| 日常開発のメイン    | 複雑な集計・特殊なSQL向き |

> Eloquentはクエリビルダを土台として利用している

## まとめ

- SQL：データベースに直接命令を書く。
- クエリビルダ：SQLをPHPらしく組み立てる道具。
- Eloquent：クエリビルダの上にあるモデルを扱う仕組み。
