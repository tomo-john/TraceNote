# パスワード設定のバリエーション

## 概要

Laravel標準で使用できるものなど

## サンプル

```php
<?php
protected function passwordRules(): array
{
    return [
        'current_password' => [
            'required',           // 必須
            'current_password',   // 現在のパスワードと一致するか
        ],

        'password' => [
            'required',           // 必須
            Password::defaults(), // 標準パスワードルール
            'confirmed'           // password_confirmと一致するか
        ],
    ];
}
```

## current_password

「現在ログインしているユーザーのパスワードと一致するか」をチェックするLaravel標準ルール。

Laravelが内部的に`Hash::check($inputPassword, Auth::user()->password);`のようなことをしてくれる。

## confirmed

フォーム:

```blade
<input wire:model="password">

<input wire:model="password_confirmation">
```

の状態として、

```php
<?php
'password' => [
    'confirmed',
]
```

と書くだけで、Laravelは勝手に`password`と`password_confirmation`を比較してくれる。

`confirmed`を書くなら、Livewireには

```php
<?php
public string $password = '';
public string $password_confirmation = '';
```

が必要。

`password_confirmation`という名前は約束事(Convention)なので、`confirm_password`などはNG。

## Password::defaults()

今のアプリの標準パスワードルールを取得するメソッド。

use宣言必要: `use Illuminate\Validation\Rules\Password;`

```php
<?php
'password' => [
    'required',
    Password::defaults(), // 標準パスワードルール
    'confirmed'
],
```

みたいに書くと、`Password::min(8)`みたいな`Password`オブジェクトが返ってくる。

`Password::defaults()` -> `new Password(...設定...)`みたいなイメージ。

### どこでdefaultsが定義されている？

`vendor/laravel/framework/src/Illuminate/Validation/Rules/Password.php`

この中に、`public static function defaults()`がある。

```php
<?php
public static function defaults($callback = null)
{
    // 引数が無ければ現在のデフォルト設定を返す
    if (is_null($callback)) {
        return static::default();
    }

    // 関数でもPasswordオブジェクトでもなければエラー
    if (! is_callable($callback) && ! $callback instanceof static) {
        throw new InvalidArgumentException(...);
    }

    // デフォルト設定として保存
    static::$defaultCallback = $callback;
}
```

今回は、引数なしの`Password::defaults()`で呼んでいるので、`return static::default();`

=> `default()`メソッドが呼ばれる

### default()メソッドは何をしている？

Laravelでは default() は、以前 defaults() で登録されたルールを返す役目。

```php
<?php
public static function default()
{
    $password = is_callable(static::$defaultCallback)
        ? call_user_func(static::$defaultCallback)
        : static::$defaultCallback;

    return $password instanceof Rule ? $password : static::min(8);
}
```

`is_callable()`とは？

=> この値は関数として実行できる？を判定するメソッド

`call_user_func()`は？

=> 渡された関数を実行するための関数

`static::$defaultCallback`には実行できる関数か値そのものが入っている。

`return $password instanceof Rule ? $password : static::min(8);`って？

=> `$password`がnullだと`static::min(8)`が実行

## では誰が最初に登録している？

Laravelスターターキット出は通常、`app/Providers/AppServiceProvider.php`で登録されている。

```php
<?php

namespace App\Providers;

use Illuminate\Validation\Rules\Password;

class AppServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        $this->configureDefaults();
    }

    protected function configureDefaults(): void
    {
        Date::use(CarbonImmutable::class);

        DB::prohibitDestructiveCommands(
            app()->isProduction(),
        );

        Password::defaults(fn (): ?Password => app()->isProduction()
            ? Password::min(12)
                ->mixedCase()
                ->letters()
                ->numbers()
                ->symbols()
                ->uncompromised()
            : null,
        );
    }
}
```

=> Laravel起動時に`Password::fefaults`してた
