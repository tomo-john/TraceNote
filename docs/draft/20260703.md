# orWhere() 配列の罠

## 概要

orWhere()に直接配列を渡すと、内側の条件が`AND`ではなく`OR`に分解されてしまう。

## 修正前のメソッド

```php
<?php
public function removeRelation(Trace $relatedTrace): void
    {
        TraceRelation::query()
            ->where([
                'from_trace_id' => $this->id,
                'to_trace_id' => $relatedTrace->id,
            ])
            ->orWhere([
                'from_trace_id' => $relatedTrace->id,
                'to_trace_id' => $this->id,
            ])
            ->delete();
    }
```

`where()`ではこうなる:

```sql
WHERE form_trace_id = ? AND to_trace_id = ?
```

`orWhere()`ではこうなる:

```sql
WHERE form_trace_id = ? OR to_trace_id = ?
```

tinkerで確認:

```php
<?php
> App\Models\TraceRelation::query()->where(['from_trace_id' => 3, 'to_trace_id' => 2])->orWhere(['from_trace_id' => 2, 'to_trace_id' => 3])->toSql()

= "select * from \"trace_relations\" where (\"from_trace_id\" = ? and \"to_trace_id\" = ?) or (\"from_trace_id\" = ? or \"to_trace_id\" = ?)"
```

## 修正後のメソッド

複数条件を`OR`でつなぐときはクロージャでグルーピングする。

```php
<?php
public function removeRelation(Trace $relatedTrace): void
{
    TraceRelation::query()
        ->where(function ($query) use ($relatedTrace) {
            $query->where('from_trace_id', $this->id)
                  ->where('to_trace_id', $relatedTrace->id);
        })
        ->orWhere(function ($query) use ($relatedTrace) {
            $query->where('from_trace_id', $relatedTrace->id)
                  ->where('to_trace_id', $this->id);
        })
        ->delete();
}
```
