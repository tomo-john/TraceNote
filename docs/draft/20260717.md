# Fortify

## 概要

ログインなどの認証機能を用意してくれるパッケージ。

## 認証の担当

`php artisan route:list`:

```bash
  GET|HEAD  login .... login › Laravel\Fortify › AuthenticatedSessionController@create
  POST      login .... login.store › Laravel\Fortify › AuthenticatedSessionController@store
```

`composer.json`:

```bash
    "require": {
        "php": "^8.3",
        "laravel/chisel": "^0.1.0",
        "laravel/fortify": "^1.37.2",
        "laravel/framework": "^13.7",
        "laravel/tinker": "^3.0",
        "livewire/flux": "^2.13.1",
        "livewire/livewire": "^4.1"
    },
```

=> Lravel Fortifyが担当している

=> Fortifyが自分でルートを登録する

## Service Provider

```bash
Laravel起動 -> Service Providerを順番に実行 -> FortifyServiceProviderも実行

=> Fortifyがloginやlogoutのルートを登録
```

なので`route/web.php`や`route/auth.php`には書いてなくても`php artisan route:list`にはちゃんと出てくる。


## FortifyServiceProvider.php

フルパス: `vendor/laravel/fortify/src/FortifyServiceProvider.php`

この中身の`namespace Laravel\Fortify;`について下記に記載。

### namespace

`namespace`はディレクトリの場所ではなく、クラスの所属名(住所) => フルパスではない。

秘密は`composer.json`にあり。

`/composer.json`:

```bash
    "autoload": {
        "psr-4": {
            "App\\": "app/",
            "Database\\Factories\\": "database/factories/",
            "Database\\Seeders\\": "database/seeders/"
        }
    },
```

このおかげで、ファイルが`app/Http/Controllers/DashboardController.php`でも、

`namespace App\Http\Controllers;`となる。

`composer.json`はFortify専用のものもある。

`vendor/laravel/fortify/composer.json`:

```bash
    "autoload": {
        "psr-4": {
            "Laravel\\Fortify\\": "src/"
        }
    },
```

`namespace Laravel\Fortify;`は実際には、`vendor/laravel/fortify/src/`となる。

### composer install

`composer install`すると、Composerがすべての`composer.json`を集めて、

`vendor/composer/autoload_psr4.php`というファイルを自動生成する。
