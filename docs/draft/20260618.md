## 概要

dbdiagram書き方など。

### usersテーブル

```php

<?php

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->string('email')->unique();
            $table->timestamp('email_verified_at')->nullable();
            $table->string('password');
            $table->rememberToken();
            $table->timestamps();
        });
        
        // 略
    }
};
```

これを`dbdiagram`で書くとこう:

```
Table users {
  id bigint [pk]
  name varchar
  email varchar [unique]
  email_verified_at timestamp
  password varchar
  remember_token varchar
  created_at timestamp
  updated_at timestamp
}
```

### リレーション

```
Ref: users.id < traces.user_id
```

これが:

```
users 1 --- N traces (1対多)
```

### 20260618作成時

```
Table users {
  id bigint [pk]
  name varchar
  email varchar [unique]
  email_verified_at timestamp [null]
  password varchar
  remember_token varchar
  created_at timestamp
  updated_at timestamp
}

Table traces {
  id bigint [pk]
  user_id bigint
  title varchar
  summary text
  content text
  status varchar [note: 'TraceStatus Enum']
  created_at timestamp
  updated_at timestamp
}

Table tags {
  id bigint [pk]
  user_id bigint
  name varchar
  color varchar [default: 'gray']
  created_at timestamp
  updated_at timestamp
}

Table tag_trace {
  id bigint [pk]
  trace_id bigint
  tag_id bigint
  created_at timestamp
  updated_at timestamp

  indexes {
    (trace_id, tag_id) [unique]
  }
}

// User - Trace
Ref: users.id < traces.user_id

// User - Tag
Ref: users.id < tags.user_id

// Trace - Tag
Ref: traces.id < tag_trace.trace_id
Ref: tags.id < tag_trace.tag_id
```
