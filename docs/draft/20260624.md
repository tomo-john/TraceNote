# クエリビルダ

## 概要

クエリビルダのメソッドが使えるものたち。

## 1. Eloquentのリレーションメソッド全般

すべてのリレーションタイプは、クエリビルダを返すので、メソッドチェーンできる。

```php
<?php
class User extends Model
{
    // 1対1
    public function profile()
    {
        return $this->hasOne(Profile::class);
    }

    // 1対多
    public function posts()
    {
        return $this->hasMany(Post::class);
    }

    // 多対1(逆方向)
    public function company()
    {
        return $this->belongsTo(Company::class);
    }

    // 多対多
    public function roles()
    {
        return $this->belongsToMany(Role::class);
    }

    // 中間テーブル経由
    public function comments()
    {
        return $this->hasManyThrough(Comment::class, Post::class);
    }

    // ポリモーフィック
    public function images()
    {
        return $this->morphMany(Image::class, 'imageable');
    }
}
```

これらすべて、`()`をつけて呼べばクエリビルダなので条件を追加できる。

```php
<?php
$user->roles()->where('name', 'admin')->first();
$user->comments()->orderBy('created_at', 'desc')->get();
$user->images()->where('is_primary', true)->first();
```

## 2. モデル自体(静的メソッド)

書き方のルール:

```php
<?php
class User extends Model
{
    public function scopeActive($query)
    {
        return $query->where('active', 1);
    }
}
```

- メソッド名の頭に`scope`を付ける
- 第一引数に`$query`(クエリビルダ)を受け取る
- 呼び出すときは`scope`を省略する

Attribute方式の書き方:

```php
<?php
use Illuminate\Database\Eloquent\Attributes\Scope;

class User extends Model
{
    #[Scope]
    protected function active($query)
    {
        return $query->where('active', 1);
    }
}
```
