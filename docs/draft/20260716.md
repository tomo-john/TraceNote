# Bladeコンポーネントの初期値

## 概要

`x-ui.input`作成時に出た疑問。

## 初期値いろいろ

```php
<?php
@props([
    'id' => 'null',
    'name',
    'type' => 'text',
    'value' => '',
])
```

### 'id' => 'null'

これはなくても動くという意味。

呼び出し側で`id`を指定しなくてもエラーでない。

### 'name'

これは「必須」という意味。

呼び出し側に`name`がないとLaravel側でエラー。

初期値を持たせない方が安全。

### 'type' => 'text'

これは、一番よく使う`text`を初期値として持たせている。

`'type' => 'null'`でも、inputタグは自動で `type="text"`としてくれるので壊れない。

ただ、コンポーネントの意図を明確にするためにtextを初期値とする。

### 'value' => ''

文字列として扱いやすいため。
