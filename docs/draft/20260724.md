# Livewireのプロパティ初期値

## 概要

「その型が、何を値として持てるか」で初期値が決まる

## PHPだけで考える

string型:

```php
<?php
public string $name;
```

この変数には必ず文字列(string)が入ります。という約束。

`$name`にはまだ何も入っていない。

=> 存在はしている、でも初期値はないという状態

この状態で、`echo $this->name;`とすると、

`Typed property must not be accessed before initialization`となってしまう。

なので、

```php
<?php
public string $name = '';
```

とすると、最初から空文字をいれておくので安心して使える。

string型なので`文字列`、空の文字列なので`''`が自然。

## User型は？

```php
<?php
public User $user;
```

この変数にはUserオブジェクトが入ります。という意味。

なので、`public User $user = '';`はUser型なのに、`''`は文字列...

=> 型が違う！となってしまう

### User型の初期値は？

```php
<?php
public User $user;

public function mount()
{
    $this->user = Auth::user(); 
}
```

みたいに`mount()`で初期値を入れる。

## nullになるのは？

```php
<?php
public ?string $name = null;
```

これは、`?string`なので`string`または`null`が許される。

## Livewireでの判断

### 入力フォーム

```php
<?php
public string $name = '';
```

これが一番自然。理由はinputは最初空だから。

### 編集画面など

```php
<?php
public string $name = '';

// mount
$this->name = $user->name;
```

最初だけ空で、すぐに`mount`で実際の名前を取る。
