# ユーザープロフィール編集画面

## 概要

スターターキット関連でどの画面がどう表示されているのか確認。

## URL

`/settings/profile` => この画面を修正したい。

`routes/setting.php`:

```php
<?php
use App\Livewire\Settings\Profile;

Route::middleware(['auth'])->group(function () {
    Route::redirect('settings', 'settings/profile');

    Route::livewire('settings/profile', Profile::class)->name('profile.edit');
});
```

`/settings/profile` => `app/Livewire/Settings/Profile`のLivewireコンポーネントを直接呼ぶ。

`App\Livewire\Settings\Profile`が起動すると、`render()`がなくても、`resources/views/livewire/settings/profile.blade.php`が自動で表示。

## 少し流れをおさらい

```bash
/settings/profileへブラウザからアクセス
    |
    |
Lravelがpublic/index.phpを起動
    |
    |
URLに対応するルートを探す
    |
    |
Route::livewire('settings/profile', Profile::class); を見つける
    |
    |- このとき、Route::middleware(['auth'])があるので、ログインしているかチェックはいる
    |
(ログイン済みなら)Profile::classがインスタンス化される
    |
    |- ここで普通ならrender()が呼ばれるけど、Livewire v3では省略できる(内部で勝手に呼ばれてる)
    |
resources/views/livewire/settings/profile.blade.php
```

## なぜprofile.blade.phpにはsidebar.blade.phpのレイアウトがあたっているのか？

この答えは、`Profile.php`にも、`profile.blade.php`にもない...

> 結論: Livewire 4では内部デフォルトが layouts.appというのは仕様

## Starter Kitが用意してくれているレイアウト

```bash
resources/views/layouts/

app.blade.php
auth.blade.php

app/
    header.blade.php
    sidebar.blade.php

auth/
    card.blade.php
    simple.blade.php
    split.blade.php
```

今回の初期設定では、`app.blade.php`の中に

```blade
<x-layouts::app.sidebar :title="$title ?? null">
```

があったので、`app/sidebar.blade.php`のレイアウトが当たってる。
