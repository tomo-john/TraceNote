# Git

## 概要

ファイル変更履歴管理システム。

GitとGitHubは別物。

## 基本

- Git: PCの中で変更履歴を管理するソフト
- GitHub: Gitのデータを置くクラウドサービス


Gitの流れ:

```
編集 -> git add -> git commit -> git push
```

### Working Directory

普段編集している場所。

まだGitは見ているだけ。

### Staging Area

`git add`すると、この変更をコミットしますと予約する場所に入る。

### Repository

`git commit`すると履歴として保存される。

### GitHub

`git push`するとGitHubにも送られる。

## add取り消し

```bash
git restore --staged
```

## コミット修正

```bash
git commit --amend
```
