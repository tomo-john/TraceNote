# 前提知識のTrace候補

## 概要

> 「前提知識として選択できるTraceは何を表示するべきか？」

## 候補

- 他ユーザーのTraceは選べない
- すでに前提知識になっているものは選べない
- 自分自身は選べない

### 自分のTraceだけ

```php
<?php
$trace = App\Models\Trace::find(2)  // CollectionのTrace

auth()->loginUsingId(1); // johnユーザー

auth()->user()->traces()->get();
```

### 自分自身を除く

```php
<?php
// どっちでもOK
auth()->user()->traces()->whereKeyNot($trace->id)->get()

auth()->user()->traces()->where('id', '!=', $trace->id)->get()
```

### すでに前提知識のものを除外

```php
<?php
$trace->prerequisiteTraces()->pluck('id') // 前提知識のidだけCollection取得
```

## 候補を取得

```php
<?php
$excludedIds = $trace->prerequisiteTraces()->pluck('id');

auth()->user()->traces()->whereKeyNot($trace->id)->whereNotIn('id', $excludedIds)->get()
```

## あらたな課題

> 前提知識を追加する候補には、「子知識としてすでに繋がっているTrace」や「関連知識として繋がっているTrace」も表示するのか？
> それらも候補から除外した方が自然なのか？
