# リレーションめも2

## 概要

`trace_relations`テーブル / `TraceRelations`モデル

## prerequisiteTraces(): 前提知識取得メソッド

```php
<?php
    public function prerequisiteTraces()
    {
        return $this->incomingRelations()
                    ->where('relation_type', 'prerequisite')
                    ->with('fromTrace')
                    ->get()
                    ->pluck('fromTrace');
    }
```

## incomingRelations(): 自分に向かってくる矢印

```php
<?php
    public function incomingRelations() :HasMany
    {
        return $this->HasMany(TraceRelation::class, 'to_trace_id');
    }
```

### 確認

- 1: PHP Array
- 2: Collection
- 3: map()
- 4: Arrow Function

| from_trace_id | to_trace_id | relation_type |
| ------------- | ----------- | ------------- |
| 1             | 2           | prerequisite  |
| 2             | 3           | child         |
| 1             | 3           | related       |
| 4             | 2           | prerequisite  |

Collectionを元に、前提知識であるTraceを取得するので、1: PHP Array と4: Arrow Functionを取得するのが目的。

=> 戻り値はTraceが格納されたCollection

### incomingRelations の()があるなしの違い

()なしはプロパティアクセス。

内部的には`incomingRelations()->get()`を実行しているので、Collectionが返る。

PHP側でCollection操作につながるので、この時点でSQLは発行済み。

()ありはメソッド呼び出し。

返ってくるのは、今回は`HasMany` => `HasMany`

```php
<?php
$this->incomingRelations()
    ->where(...)
    ->with(...)
    ->orderBy(...)
```

などが使用できる。=> この段階ではまだSQLが発行されていない。

`get()`した時にSQLが発行される。

### with()

```php
<?php
->with('fromTrace')
```

=> あとでfromTraceを使うから、まとめて取っておいてねという予約みたいなもの

=> SQLはまだ発行されていない
