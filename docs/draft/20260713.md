# Webサーバー

## 概要

ブラウザからのリクエストを受け取ってレスポンスを返すソフトウェア。

## 有名なWebサーバー

- Apache
- Nginx(エンジンエックス)
- IIS(Windows)

## Laravelの場合

LaravelはPHPで作られている。

でもNginxはPHPを実行できない。

```bash
ブラウザ -> Nginx -> PHP-FPM -> Laravel
```

## PHP-FPM

PHPを実行する専用プログラム。

=> 静的ファイルはNginxが返す

## php artisan serve

この場合はPHPに組み込まれている開発用Webサーバーが動いている。

なのでNginxは使われていない。

あくまで開発用サーバーなので、本番環境では使われない。

## Nginx検証

Ubuntuにインストール:

```bash
sudo apt install nginx
```

=> パッケージインストール後に自動で起動する

Nginx起動確認:

ブラウザで`http://localhost`を開いて、「Welcome to nginx」を見る。

`/var/www/html/index.nginx-debian.html`が自動で生成されている。

初期状態では`root`しか編集ができない。

### Nginxの設定ファイル

`/etc/nginx/sites-available/default`

ここに、

```bash
root /var/www/html;
```

```bash
server {
	listen 80 default_server;
	listen [::]:80 default_server;
```

など、今Nginxがどう動いているかの設定が書いてある。
