# Webページが表示されるまでおさらい

## 概要

サービスはどうやって80番ポートを開いて、ブラウザからアクセスを受け付けるのか？

## 全体の流れ(一例)

ブラウザで、`http://160.xxx.xxx.xxx`を開くと...

```bash
ブラウザ => TCPで80番ポートへ接続 => Ubuntuカーネル

=> nginx(サービス) => index.html => nginx => ブラウザへHTMLを返す
```

### ブラウザ

- `http://` = `TCP 80`
- `https://` = `TCP 443`

### Ubuntuカーネル

ネットワークカードに届いた通信はまずLinuxカーネルが受け取る。

ここで、「80番？誰か使ってる？」と確認。

もし誰も80番を使っていなければ(ポートが閉じられていれば)`Connection refused`になる。

### Nginxが80番を開く

Nginxを起動:

```
sudo systemctl start nginx
```

=> systemdが`/usr/sbin/nginx`を起動する。

するとnginxは「80番を使います！」とLinuxカーネルにお願いする。

=> これを、`bind()`という

80番ポートを開いているのはLinuxでなくnginx(webサーバー)。

### 通信が届く

ブラウザから`GET /`が届くと、nginxは設定ファイル`/etc/nginx/sites-enabled/default`を見る。

もし、`root /var/www/html;`なら`/var/www/html/index.html`を探す。

### index.htmlを読む

`index.html`の内容をメモリへ読み、ブラウザへ返す。

### Laravelの場合

NginxはPHPを実行することはできないので、`PHP-FPM`へ「このPHPお願い」と仕事を渡している。
