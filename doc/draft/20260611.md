# Wireshark 基礎まとめ 

## Wiresharkとは

Wiresharkは、ネットワーク上を流れる通信(パケット)を取得し、内容を解析できるソフト。

一言でいうと、

> ネットワーク通信を見える化するツール

---

## パケットとは

パケット(Packet)とは、ネットワーク上で送受信されるデータの単位。

例えば、

* Webサイト閲覧
* メール送信
* ping
* SSH接続

などの通信は、すべてパケットとしてやり取りされている。

---

## Wiresharkでできること

* 通信内容の確認
* パケットの取得(Capture)
* プロトコルの解析
* 通信トラブルの調査
* ネットワーク学習

---

## 主なプロトコル

### ARP

IPアドレスからMACアドレスを調べる。

### ICMP

pingなどで使用される。

### DNS

ドメイン名からIPアドレスを調べる。

例:

* google.com
* github.com

### TCP

信頼性の高い通信を行う。

3-way handshakeで接続を確立する。

### UDP

高速な通信を行う。

動画配信や音声通信などで利用される。

### HTTP

Webページの通信。

### HTTPS

HTTPをTLSで暗号化したもの。

---

## Wiresharkの基本的な流れ

1. インターフェースを選択
2. Capture開始
3. 通信を発生させる
4. パケットを確認
5. Capture停止

---

## よく使うフィルタ

DNSのみ表示: `dns`

TCPのみ表示: `tcp`

HTTPのみ表示: `http`

HTTPSのみ表示: `tls`

ICMPのみ表示: `icmp`

特定IPのみ表示: `ip.addr == 192.168.1.10`

---

## 実際に見える通信

Webサイトへアクセスすると、

```
DNS => TCP 3-way handshake => TLS => HTTP GET => レスポンス受信
```

という流れを見ることができる。

---

## よくある用途

* 通信できない原因調査
* DNS障害の確認
* パケットロスの調査
* TCP再送の確認
* ネットワーク学習

---

## Wiresharkは何のために使うか

「アプリケーションの裏側で何が起きているか」を確認するためのツール。

ネットワーク通信を可視化し、問題解析や学習に役立てることができる。
