### 1. カッコの「あり」「なし」で大違い

`$user->traces()`: カッコあり
- `HashMany`クラス(クエリビルダの一種)
- DBデータ取得前の状態(SQL文の条件を組み立ている途中のオブジェクト
- さらに条件を絞り込みたいときなどに使用する
    - `$user->traces()->where('status', 'draft')`みたいに

`$user->traces`: カッコなし
- `Collection`(コレクション)
- DBにSQLを発行し、データをすべて取得し終わって、配列風オブジェクト(Collection)にされている
- データをそのままループで回したり、`fillter()`したいときとかに使用する

> つまり、$user->tracesは、裏側で $user->traces()->get() を勝手に実行してくれている。

この`$user->traces`(カッコなし)を呼び出した瞬間に、Laravelは全データを取得する。

=> Traceのタイトルや本文など全てのデータを一度メモリに読込む。

=> その上でPHPがコレクション機能でグループ分けとカウントをしている。

=> データの数が何千件と増えてくるとメモリへの負担が大きくなる...

### DB(SQL)側で一発集計

すべてをクエリビルダ(カッコあり)に統一して、DB側に集計を丸投げする。

```php
<?php
// PHPクラスで実施

$statusCount = $user->traces()
    ->selectRaw('status, count(*) as count')
    ->groupBy('status')
    ->pluck('count', 'status')
    ->toArray();
```
