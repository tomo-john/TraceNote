# DI(依存性注入)

## 概要

ServiceをControllerで使う時のメモ

### Service

こんなService:

```php
<?php

namespace App\Services;

use App\Models\User;
use App\Models\Trace;
use App\Models\Tag;
use App\Enums\TraceStatus;

class DashboardService
{
    public function getStats(User $user): array
    {
        return [
            'traceCount' => $user->traces()->count(),
            'tagCount' => $user->tags()->count(),
        ];
    }
}
```

これを`app/Http/Controllers/DashboardController.php`で使いたい。

`getStats`は`static`でないのでインスタンスの作成が必要。

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Services\DashboardService;

class DashboardController extends Controller
{
    public function __invoke()
    {
        $user = auth()->user();

        $service = new DashboardService();

        $stats = $service->getStats($user);

        return view('dashboard', $stats);
    }
}
```

とりあえず`new DashboardService()`でインスタンスを作ってあげたらいける。

しかし、これはController側がServiceの内部構造を気にしなければならなくなる。

=> コードが複雑化(結合度)が上がってしまう

そこで登場するのが、Laravelの機能の1つである依存性の注入(`DI: Dependency Injection`)

### DI

Controller側のメソッド(またはコンストラクタ)の引数に、このクラスが使いたいと型とヒントだけ書く。

すると、Laravelが裏側で自動的に`new DashboardService()`を実行して完成したインスタンスをくれる。

```php
<?php

class DashboardController extends Controller
{
    public function __invoke(DashboardService $service)
    {
        $user = auth()->user();

        $stats = $service->getStats($user);

        return view('dashboard', $stats);
    }
}
```

今回のような単一のアクションコントローラー(`__invoke`)の場合は、引数に仕込むのがスマート(らしい)
