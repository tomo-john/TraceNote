# Controller基礎

## 単一アクションコントローラー

`__invoke`(インボーク)とは、PHPが最初から持っている魔法のメソッド。(マジックメソッド)

クラスのインスタンスを関数みたいに直接実行したときに自動で呼び出されるメソッド。

### マジックメソッドとは？

PHPには、特定の条件を満たした時にブラウザやPHPが裏側で自動で実行してくれる特殊なメソッドがある。

=> これをマジックメソッドと呼ぶ

必ず頭にアンダースコアが2つ(`__`)つくのがルール。

クラスが新しく作成された(`new`された)ときに自動で動く、`__construct()`などもそれ。

`__invoke()`は、オブジェクトの変数を関数っぽく`()`をつけて呼び出した瞬間に裏側で自動でトリガーされる。

```php
<?php
class Dog {
    // クラスの中に関数みたいに呼び出されたときの処理を書いておく
    public function __invoke() {
        return "ワン！";
    }
}

$john = new Dog(); // インスタンス化

// 普通なら $john->bark() のようにメソッドを呼び出しますが...
// オブジェクトに直接 () をつけて、関数みたいに実行してみる
echo $john(); 

// 出力： "ワン！" (裏で __invoke が勝手に動く)
```
